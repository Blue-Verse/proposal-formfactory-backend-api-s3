핵심 요구사항 분석

  • RDBMS 스키마 설계·영구 구축 — 주문·단가표·설정 등 기존 LocalStorage 데이터 모델을 정규화하여 영속 저장소로 이관
  • LocalStorage 트랜잭션 100% 대체 REST API — 기존 화면 단 로직(UI/UX)을 보존한 채 fetch() 호출 지점에 1:1 매핑되는 API 엔드포인트 설계
  • S3 Presigned URL 다이렉트 업로드 — 블랙박스 등 1회 최대 2GB 대용량 파일을 서버 경유 없이 클라이언트→S3 직접 전송
  • 관리자 마스터 대시보드 API + 인증(Auth) — 통계·제어 API 및 인증/인가 로직 구축
  • 비즈뿌리오 알림톡/SMS 자동 발송 — 외부 API 연동 기반 이벤트 트리거 발송
  • 3대 코어 보안 로직 — UUID v4 기반 브루트포스/Key Collision 방어, 웹쉘 업로드 차단, 페이로드 과부하 방지

제안 포인트

1) "프론트엔드 0원" 제약을 정확히 이해한 순수 백엔드 제안. 기획·디자인·퍼블리싱 공수를 일절 산정하지 않고, 제공된 Vanilla JS 소스의 fetch() 통신 규약에 맞춰 API 계약(요청/응답 스키마)을 설계합니다. React/Vue 전환 제안 없이 기존 화면 로직을 그대로 보존합니다.

2) Presigned URL + UUID v4 난수화는 검증된 구현 패턴. 유사 프로젝트에서 S3 호환 스토리지의 Presigned URL 다이렉트 업로드와 파일명 난수화(Key Collision 방어)를 운영 수준으로 구현한 경험이 있어, 2GB 대용량 멀티파트 업로드까지 안정적으로 처리합니다.

3) 3대 보안 로직은 기존에 구축해 본 영역. 브루트포스 방어, 업로드 파일 검증(웹쉘 차단), Rate Limiting/페이로드 크기 제한(과부하 방지)을 엔터프라이즈 결재 시스템에서 다계층 보안으로 구현한 경험을 그대로 적용합니다.

4) 알림톡/SMS는 동일 제공사(Ppurio/비즈뿌리오) 연동 경험 보유. 발송 큐와 이벤트 트리거 기반 자동 발송 구조를 다수 프로젝트에서 운영해, 연동 리스크가 낮습니다.

5) 산출물 완성도(1순위)에 집중. 다수 인력으로 단계를 병행하여 30일 일정을 유지하면서, ERD·Swagger/Postman 명세·배포 가이드까지 인수인계 가능한 형태로 완비합니다.

PRD (Product Requirements Document)

과업지시서(RFP) 기반으로 백엔드 기능 범위를 정의하고 구현 기준을 명확히 합니다.

1. 프로젝트 배경 및 목적

행정사사무소용 스마트 서면 설계 플랫폼 FORMFACTORY는 프론트엔드(HTML/CSS/Vanilla JS)가 100% 완성되어 있으나, 데이터가 브라우저 LocalStorage에 저장되어 영속성·다중 사용자·대용량 파일 처리에 한계가 있습니다. 본 과업의 목적은 LocalStorage 트랜잭션을 완전히 대체하는 REST API 서버와 RDBMS, 그리고 대용량 파일을 위한 S3 클라우드 아키텍처를 신규 구축하여, 기존 화면을 변경 없이 그대로 운영 가능한 정식 서비스로 전환하는 것입니다.

3. 기능 요구사항

3.1 데이터베이스 (RDBMS)

  • [필수] 주문·단가표·설정 등 핵심 도메인 스키마 설계 및 영구 구축
  • [필수] LocalStorage 데이터 구조를 정규화하여 무손실 이관 가능하도록 모델링
  • [필수] ERD 산출물 제공

3.2 REST API 서버

  • [필수] 기존 LocalStorage 트랜잭션을 100% 대체하는 API 엔드포인트
  • [필수] 기존 Vanilla JS의 fetch() 호출과 1:1 바인딩 (요청/응답 스키마 정합)
  • [필수] 기존 화면 단 로직(UI/UX) 100% 보존 — 프레임워크 전환 없음

3.3 S3 대용량 파일 업로드

  • [필수] Presigned URL 방식 클라이언트→S3 다이렉트 업로드 (서버 부하 분산)
  • [필수] 1회 최대 2GB(블랙박스 등) 대용량 파일 처리
  • [필수] UUID v4 기반 파일명 난수화 (Key Collision 방어)

3.4 관리자 대시보드 & 인증

  • [필수] 마스터 대시보드 통계·제어 API
  • [필수] 인증(Auth) 및 권한 분리 로직
  • [필수] 50MB 페이로드 과부하 방지를 위한 경량 트리 메타데이터 응답 설계

3.5 외부 연동

  • [필수] 비즈뿌리오 등 외부 API 기반 알림톡/SMS 자동 발송 트리거

3.6 보안 (3대 코어 로직)

  • [필수] UUID v4 기반 브루트포스/추측 방어
  • [필수] 웹쉘 업로드 차단 (확장자/MIME/매직넘버 검증)
  • [필수] 페이로드 과부하 방지 (요청 크기 제한 + Rate Limiting)

4. 비기능 요구사항

항목요구사항

성능대용량 파일은 서버 미경유(다이렉트 업로드), 대시보드 응답은 경량 메타데이터로 과부하 방지

5. 기술 제약사항

  • 제공된 기존 소스코드(Vanilla JS)의 fetch() 통신과 1:1 바인딩 — React/Vue 등 프레임워크 전환 제안 불가
